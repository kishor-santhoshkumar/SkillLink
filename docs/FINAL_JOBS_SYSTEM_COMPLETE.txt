================================================================================
API ENDPOINTS:
================================================================================

JOBS:
POST   /jobs/                          - Create job (company only)
GET    /jobs/                          - Get jobs (filtered by role)
POST   /jobs/{id}/apply                - Apply to job (worker only)
GET    /jobs/{id}/applications         - Get applicants (company only)
PATCH  /jobs/applications/{id}/accept  - Accept applicant (company only)

NOTIFICATIONS:
GET    /notifications/                 - Get all notifications
GET    /notifications/unread-count     - Get unread count
PATCH  /notifications/{id}/read        - Mark as read
PATCH  /notifications/mark-all-read    - Mark all as read

RESUMES (UPDATED):
POST   /resumes/                       - Create profile (AUTH REQUIRED)
POST   /resumes/structured             - Create from form (AUTH REQUIRED)

HOW TO USE:
================================================================================

1. START BACKEND:
   > restart_backend.bat

2. START FRONTEND:
   > restart_frontend.bat

3. IF PROFILE LINKING ISSUES:
   > python link_resumes_to_users.py
   (Links existing profiles to users)

4. COMPANY WORKFLOW:
   - Login as company
   - Go to Jobs page
   - Click "Post New Job"
   - Fill form and submit
   - Wait for applications
   - Click "View Applicants" on job card
   - Click "View Profile" to see full worker profile
   - Click "Accept" to hire worker
   - Worker gets notification

5. WORKER WORKFLOW:
   - Login as worker
   - Create profile (if not exists)
   - Go to Jobs page
   - Browse available jobs
   - Click "Apply Now"
   - Company gets notification
   - Wait for acceptance
   - Check bell icon for notification
