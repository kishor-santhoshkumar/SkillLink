FEATURE: Automatic Email Notification via n8n Webhook
When a worker applies to a job, the system now:
1. Saves application to database
2. Creates in-app notification
3. Triggers n8n webhook to send email to company

FLOW DIAGRAM
================================================================================

Worker clicks "Apply Now"
         ↓
Frontend: POST /jobs/{job_id}/apply
         ↓
Backend performs 3 actions:
         ↓
    ┌────┴────┬────────────┬──────────────┐
    ↓         ↓            ↓              ↓
Save to DB  Create      Trigger n8n    Return
            Notification  Webhook       Success
    ↓         ↓            ↓              ↓
job_        notifications  →Email sent   Frontend
applications table        to company    shows success

================================================================================
DATABASE STRUCTURE
================================================================================

Table: job_applications (already exists)
- id (Primary Key)
- job_id (Foreign Key → job_requests)
- worker_id (Foreign Key → resumes)
- status (pending/accepted/rejected)
- created_at
- updated_at

================================================================================
ERROR HANDLING
================================================================================

✓ Duplicate application check
  → Returns: "You have already applied for this job"

✓ Job not found
  → Returns: "Job not found"

✓ Job closed
  → Returns: "Job is not open for applications"

✓ No worker profile
  → Returns: "Please create your profile first"

✓ Webhook failure
  → Application still saved
  → Error logged but doesn't fail the request
  → User sees success message
