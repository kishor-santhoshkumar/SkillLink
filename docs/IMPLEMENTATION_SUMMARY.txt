   Function: apply_to_job()
   Location: Line 520-630
   
   Changes:
   - Added code to fetch company user and company profile from database
   - Build complete company_data dict with name, email, location
   - Pass company_data to webhook function instead of just email
   - Improved logging to show company email being used
   
   New Code:
   ```python
   # Get company user and company profile for email and details
   company_user = db.query(User).filter(User.id == job.company_id).first()
   company = db.query(Company).filter(Company.user_id == job.company_id).first()
   
   company_email = company_user.email if company_user and company_user.email else "no-email@example.com"
   
   # Build complete company data
   company_data = {
       "company_name": company.company_name if company else company_user.username,
       "email": company_email,
       "location": company.location if company else job.location
   }
   
   # Trigger webhook with complete data
   status_msg, response_code, response_body, error_msg = await trigger_n8n_webhook(
       worker_data, job_data, company_data
   )
   ```

DATA FLOW
================================================================================

1. Worker applies to job
   ↓
2. Backend receives /jobs/{job_id}/apply request
   ↓
3. Fetch Job from database (get company_id)
   ↓
4. Fetch Company User from database (get email)
   ↓
5. Fetch Company Profile from database (get name, location)
   ↓
6. Build webhook payload with DYNAMIC company email
   ↓
7. Trigger n8n webhook with complete data
   ↓
8. n8n receives webhook with correct company email
   ↓
9. n8n Gmail node sends email to company_email (DYNAMIC)
   ↓
10. Company receives notification for their job application
