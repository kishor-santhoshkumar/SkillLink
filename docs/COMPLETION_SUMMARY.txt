================================================================================
HOW IT WORKS NOW
================================================================================

1. Worker applies to job
   ↓
2. Backend receives /jobs/{job_id}/apply request
   ↓
3. Fetch Job from database (get company_id)
   ↓
4. Fetch Company User from database (get email)
   ↓
5. Fetch Company Profile from database (get name, location)
   ↓
6. Build webhook payload with DYNAMIC company email
   ↓
7. Trigger n8n webhook with complete data
   ↓
8. n8n receives webhook with correct company email
   ↓
9. n8n Gmail node sends email to company_email (DYNAMIC)
   ↓
10. Company receives notification for their job application

SUPPORT & TROUBLESHOOTING
================================================================================

ISSUE: Company not receiving emails
SOLUTION:
1. Check if company email is set in User table
2. Check backend logs for webhook trigger status
3. Check n8n webhook logs for errors
4. Verify n8n Gmail node configuration

ISSUE: Wrong company receiving email
SOLUTION:
1. Verify job.company_id is correct
2. Check User table for correct email mapping
3. Review ApplicationLog for webhook payload

ISSUE: Missing worker email in notification
SOLUTION:
1. Verify worker User.email is set
2. Check if worker profile was created
3. Review webhook payload in logs

ISSUE: Webhook not triggered
SOLUTION:
1. Check backend logs for errors
2. Verify job.company_id is correct
3. Verify company_user exists in database
4. Check if company email is set

For more troubleshooting, see WEBHOOK_PAYLOAD_REFERENCE.txt
