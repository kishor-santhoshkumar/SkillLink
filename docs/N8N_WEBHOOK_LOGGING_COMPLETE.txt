DATABASE SCHEMA
================================================================================

Table: application_logs
-----------------------
id                      SERIAL PRIMARY KEY
worker_id               INTEGER (FK → resumes)
job_id                  INTEGER (FK → job_requests)
application_id          INTEGER (FK → job_applications)
email_trigger_status    VARCHAR(100) - Status message
webhook_response_code   INTEGER - HTTP response code (200, 404, 500, etc.)
webhook_response_body   TEXT - Response from n8n
error_message           TEXT - Error details if failed
created_at              TIMESTAMP - When webhook was triggered

================================================================================
WORKFLOW WHEN WORKER APPLIES
================================================================================

Step 1: Worker clicks "Apply Now"
         ↓
Step 2: Backend validates application
         ↓
Step 3: Save to job_applications table
         ↓ 
         📝 LOG: "Worker application saved to database"
         ↓
Step 4: Create in-app notification
         ↓
         📝 LOG: "Company dashboard receives application"
         ↓
Step 5: Trigger n8n webhook
         ↓
         📝 LOG: "n8n webhook triggered"
         ↓
Step 6: Evaluate webhook response
         ↓
         ✅ Success (200): "Webhook triggered successfully"
         ⚠️  Warning (4xx/5xx): "Webhook returned status XXX"
         ❌ Error: "Webhook timeout" or "Webhook failed"
         ↓
         📝 LOG: "Email workflow started - Status: [status]"
         ↓
Step 7: Save to application_logs table
         ↓
Step 8: Return response to frontend
         {
           "message": "Application submitted successfully",
           "application_id": 123,
           "email_trigger_status": "Webhook triggered successfully"
         }

================================================================================
API RESPONSE
================================================================================

Success Response:
{
  "message": "Application submitted successfully",
  "application_id": 123,
  "email_trigger_status": "Webhook triggered successfully"
}

Possible email_trigger_status values:
- "Webhook triggered successfully" (200 OK)
- "Webhook returned status 404" (n8n workflow not found)
- "Webhook returned status 500" (n8n internal error)
- "Webhook timeout" (n8n didn't respond in 10 seconds)
- "Webhook failed" (network error or other exception)

================================================================================
ERROR HANDLING
================================================================================

Scenario 1: n8n workflow not activated
- Response Code: 404
- Status: "Webhook returned status 404"
- Application: Still saved ✅
- Notification: Still created ✅
- Email: Not sent ❌

Scenario 2: n8n timeout
- Response Code: None
- Status: "Webhook timeout"
- Application: Still saved ✅
- Notification: Still created ✅
- Email: Unknown (may or may not be sent)

Scenario 3: Network error
- Response Code: None
- Status: "Webhook failed"
- Error Message: Detailed error
- Application: Still saved ✅
- Notification: Still created ✅
- Email: Not sent ❌

IMPORTANT: Application is ALWAYS saved even if email fails!
