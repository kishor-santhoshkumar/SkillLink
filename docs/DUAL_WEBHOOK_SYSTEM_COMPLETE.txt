================================================================================
ERROR HANDLING
================================================================================

Both webhooks have robust error handling:

✓ Application/Acceptance always saved even if email fails
✓ In-app notifications work independently
✓ Webhook failures are logged but don't break flow
✓ Fallback values for missing data
✓ Timeout protection (10 seconds)
✓ Detailed error messages for debugging

Common Scenarios:
1. Webhook not activated → 404 error (logged, app continues)
2. Network timeout → Logged, app continues
3. Missing email → Uses fallback "no-email@example.com"
4. Missing company data → Uses job data as fallback
