PROBLEM STATEMENT
================================================================================

Previously, when a worker applied for a job, the webhook always sent notifications
to a hardcoded email address (230697.ad@rmkec.ac.in), regardless of which company
posted the job.

This was incorrect because:
- Different companies should receive notifications for their own job applications
- The hardcoded email doesn't represent the actual company
- No personalization or proper routing of notifications

================================================================================
SOLUTION IMPLEMENTED
================================================================================

1️⃣ DYNAMIC COMPANY EMAIL FETCHING
   Location: app/routers/job.py - apply_to_job() function (line ~580)
   
   Process:
   a) When worker applies to job:
      - Fetch job from database using job_id
      - Get company_id from job record
      - Query User table using company_id to get company's registered email
      - Use company_user.email as the notification recipient
   
   Code:
   ```python
   company_user = db.query(User).filter(User.id == job.company_id).first()
   company_email = company_user.email if company_user and company_user.email else "no-email@example.com"
   ```

2️⃣ ENHANCED WEBHOOK PAYLOAD
   Location: app/routers/job.py - trigger_n8n_webhook() function (line ~35)
   
   Updated payload now includes ALL required fields:
   - worker_name: Full name of the worker
   - worker_email: Email of the worker (for future confirmations)
   - trade: Worker's primary trade/skill
   - experience: Years of experience
   - location: Worker's location
   - job_title: Job title/required trade
   - company_name: Name of the company
   - company_email: DYNAMIC email from database (NOT hardcoded)
   - company_location: Company's location
   
   Example payload:
   ```json
   {
     "worker_name": "John Doe",
     "worker_email": "john@example.com",
     "trade": "Carpenter",
     "experience": 5,
     "location": "Mumbai, Maharashtra",
     "job_title": "Carpenter",
     "company_name": "ABC Construction",
     "company_email": "company@example.com",
     "company_location": "Mumbai"
   }
   ```

3️⃣ COMPANY DATA RETRIEVAL
   Location: app/routers/job.py - apply_to_job() function (line ~600)
   
   Before triggering webhook, the system now:
   - Queries User table for company email
   - Queries Company table for company details (name, location)
   - Builds complete company_data dictionary
   - Passes to webhook function
   
   Code:
   ```python
   company_user = db.query(User).filter(User.id == job.company_id).first()
   company = db.query(Company).filter(Company.user_id == job.company_id).first()
   
   company_data = {
       "company_name": company.company_name if company else company_user.username,
       "email": company_email,
       "location": company.location if company else job.location
   }
   ```

4️⃣ WEBHOOK FUNCTION SIGNATURE UPDATE
   Old: trigger_n8n_webhook(worker_data, job_data, company_email)
   New: trigger_n8n_webhook(worker_data, job_data, company_data)
   
   Reason: Passing complete company_data dict instead of just email
   allows for more flexibility and includes all company information

================================================================================

        ↓

        ↓
Fetch Job from database (get company_id)
        ↓
Fetch Company User from database (get email)
        ↓
Fetch Company Profile from database (get name, location)
        ↓
Build webhook payload with DYNAMIC company email
        ↓

        ↓
n8n receives webhook with correct company email
        ↓

        ↓
