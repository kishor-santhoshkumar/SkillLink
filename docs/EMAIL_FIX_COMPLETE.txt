✅ EMAIL ISSUE FIXED - COMPLETE
================================================================================

PROBLEM:
Both worker and company were receiving emails at the same address 
(230697.ad@rmkec.ac.in) instead of their individual login emails.

ROOT CAUSE:
The accept_application endpoint was checking Company.email first, then falling
back to User.email. This caused it to use the wrong email from the companies
table instead of the login email from the users table.

SOLUTION:
Updated the code to ONLY use User.email (the email from registration/login)
for ALL webhook notifications.

================================================================================
WHAT WAS CHANGED
================================================================================

File: app/routers/job.py
Function: accept_application()

BEFORE:
-------
company_data = {
    "email": company.email if company and company.email else current_user.email
}
# This checked Company.email first ❌

AFTER:
------
company_data = {
    "email": current_user.email if current_user.email else "no-email@example.com"
}
# Now ONLY uses User.email ✅

================================================================================
HOW IT WORKS NOW
================================================================================

Webhook 1: Worker Application → Company
----------------------------------------
Email sent to: Company's User.email (login email) ✅

Webhook 2: Job Acceptance → Worker
-----------------------------------
Worker email: Worker's User.email (login email) ✅
Company contact: Company's User.email (login email) ✅

Both webhooks now use ONLY the users table email.
The companies table email is ignored.

================================================================================
EMAIL SOURCES
================================================================================

✅ USED: users.email
   - Email from registration/login
   - Used for all webhooks
   - Each user has unique email

❌ NOT USED: companies.email
   - Just profile information
   - Not used for webhooks
   - Can be different from login email
